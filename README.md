# AWS Elastic Load Balancer overview

This project demonstrates how to configure and use AWS Elastic Load Balancing (ELB) to distribute incoming traffic across multiple **Amazon EC2 instances.
The goal of this project is to ensure high availability, scalability, and fault tolerance for a web application.

How to creat;

1) Two or more EC2 instances are launched.

2) A simple web server (like **Apache HTTP Server or **Nginx) is installed on each instance.

3) An Elastic Load Balancer is created.

4) EC2 instances are registered as targets in the Load Balancer.

5) The Load Balancer distributes incoming HTTP requests across all instances.

 # Why its important

1) Using a Load Balancer provides several advantages:

2) Improves application availability

3) Distributes traffic evenly across servers

4) Prevents server overload

5) Supports automatic scaling

6) Provides fault tolerance

Load balancing is a key concept used in cloud computing and DevOps architectures.
